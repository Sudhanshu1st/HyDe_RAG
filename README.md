Secure HyDe RAG: Advanced Document Retrieval System

📖 Overview

Secure HyDe RAG is an in-house Retrieval-Augmented Generation (RAG) application designed for querying complex technical documentation (SOPs, RCA-CAPA, Engineering Manuals) with high precision.

Unlike standard RAG systems that search using raw user queries, this project implements HyDe (Hypothetical Document Embeddings). It generates a "hallucinated" hypothetical answer to the user's question, embeds that hypothesis, and uses it to find semantically similar real documents. This approach significantly improves retrieval accuracy for technical jargon and domain-specific questions.

Crucially, this system is built to run with open-source models via Hugging Face APIs, offering a template for secure, internal deployment without relying on closed-source, data-harvesting AI platforms.

🚀 Key Features

HyDe Implementation: Uses "Hypothetical Document Embeddings" to bridge the gap between short queries and dense technical documents.

Privacy-First Design: Designed to work with open-source models (e.g., Flan-T5, Mistral), ensuring data control.

Vector Search: Utilizes FAISS for efficient, local similarity search.

Streamlit UI: A clean, interactive web interface for uploading PDFs and visualizing the RAG pipeline.

Transparent Logic: Visualizes the "Hypothetical Document" generated by the AI vs. the "Real Source" retrieved from the file.

🛠️ How It Works (The HyDe Logic)

Standard RAG searches for: Query <-> Document similarity.

HyDe RAG searches for: Hypothesis <-> Document similarity.

User Query: "How do I recalibrate the pressure sensor?"

Step 1 (Generate): The LLM hallucinates a hypothetical answer: "To recalibrate the sensor, access the maintenance menu, select calibration, and apply zero-reference gas..."

Step 2 (Embed & Retrieve): We embed this hallucination. It captures the semantic meaning of answering the question. We use this vector to search the real PDF.

Step 3 (Synthesize): We retrieve the actual manual page and generate the final, factually correct answer.

📦 Tech Stack

Language: Python

Framework: LangChain

Frontend: Streamlit

LLM Provider: Hugging Face Inference API (Free Tier compatible)

Models: * Generation: google/flan-t5-large (or Mistral-7B)

Embeddings: sentence-transformers/all-MiniLM-L6-v2

Vector Store: FAISS (Facebook AI Similarity Search)

⚙️ Installation & Setup

Clone the Repository

git clone [https://github.com/yourusername/secure-hyde-rag.git](https://github.com/yourusername/secure-hyde-rag.git)
cd secure-hyde-rag


Install Dependencies

pip install -r requirements.txt


Configure API Key

Get a free API Access Token from Hugging Face.

Open app.py and replace the placeholder:

hf_api_key = "hf_xxxxxxxxxxxxxxxxxxxxxxxx"


▶️ Usage

Run the Application

streamlit run app.py


Interact

Open your browser to the local URL (usually http://localhost:8501).

Upload a PDF (e.g., a technical manual or research paper).

Wait for the "Processed!" message.

Ask a question.

Expand the "View Generated Hypothetical Document" section to see the HyDe magic in action!
